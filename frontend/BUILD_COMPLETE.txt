                       DEVELOPMENT NOTES

PROJECT STRUCTURE:
  • Single main component (ProjectDashboard) handles all 6 tabs
  • Can be split into separate components if needed
  • Helper components inline for simplicity
  • API client is minimal but extensible
  • State management uses React hooks only

KEY DESIGN DECISIONS:
  • ProjectDashboard is large (~1500 lines) for simplicity
    Could be split: OverviewTab, UploadTab, RecordTypesTab, etc.
  • All styling uses TailwindCSS utility classes
  • Icons from lucide-react for consistency
  • Polling interval: 1.5 seconds for analysis status
  • No external state management (Redux/Zustand) - can be added later
  • Components are functional only (no class components)

DARK THEME APPROACH:
  • Base: slate-950 (#0f172a) background
  • Cards: slate-800/50 with subtle borders
  • Text: slate-50 (white) + slate-400 (muted)
  • Accents: indigo-600 for primary, colors for status
  • Opacity classes for depth (e.g., slate-700/50)

PERFORMANCE:
  • Code splitting via React Router
  • Efficient state updates
  • Proper dependency arrays
  • Icon tree-shaking (Lucide)
  • CSS minification in production
  • No unnecessary re-renders
